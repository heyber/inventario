Base de datos: PostgreSQL corriendo en Docker
ORM: Prisma conectado a la DB

🐳 Docker

Uso docker-compose con 2 servicios:

app: corre Next.js
db: PostgreSQL

La conexión a la base de datos es:

postgresql://postgres:postgres@db:5432/mydb

🧩 Prisma

Uso schema.prisma
Ya tengo configurado el generator client
Uso npx prisma generate y npx prisma db push

Modelo de usuario:

model User {
id String @id @default(cuid())
name String?
email String @unique
password String

city String?
company String?
phone String?

createdAt DateTime @default(now())

products Product[]
inventories Inventory[]
inventoryFiles InventoryFile[]
}

Modelo de archivo de inventario (principal actual):

model InventoryFile {
id String @id @default(cuid())
name String
description String?
data Json
userId String
createdAt DateTime @default(now())

user User @relation(fields: [userId], references: [id])
}

Nota:
El modelo Inventory quedó como modelo antiguo para filas individuales,
pero el sistema actual trabaja con InventoryFile (archivos completos de Excel).

🖥️ Frontend

Componentes principales:

InventoryUploadBox
→ Lee el Excel y envía data al page

InventoryTable
→ Lista los archivos guardados
→ Botones de ver y eliminar

InventoryModal
→ Muestra el contenido completo del Excel

Flujo actual:

Upload Excel
→ InventoryUploadBox
→ page.tsx (fetch API)
→ /api/inventory-file/create
→ PostgreSQL

Luego:

useEffect
→ /api/inventory-file/list
→ InventoryTable

Acción “Ver”
→ abre InventoryModal
